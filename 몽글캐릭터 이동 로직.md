# 몽글 캐릭터 이동 로직 — 한 줄씩 코드 해석

대상 파일: `MongleFeatures/Sources/MongleFeatures/Design/Components.swift`
대상 타입: `MongleCharacter` (L1138~1161), `MongleSceneView` (L1323~1556)

---

## 0. 한눈에 보는 시뮬레이션 흐름

```
┌──────────────┐                ┌──────────────────────────────┐
│ initMongles  │ ── 초기 배치 ──▶ │ mongles: [MongleCharacter]   │
└──────────────┘                └──────────────┬───────────────┘
                                               │
                          0.16s 마다           ▼
                                  ┌────────────────────────┐
                                  │  step(size:)            │
                                  │  ─ 휴식 처리             │
                                  │  ─ 목표 도달 판정         │
                                  │  ─ 이동(단위벡터·stepSize)│
                                  │  ─ 벽 감지(clamp+재타겟)  │
                                  │  ─ 충돌 감지(정지+카운트)  │
                                  │  ─ 정상 이동 + hopY 갱신  │
                                  └────────────────────────┘
                                               │
                                               ▼
                          @State mongles 변경 → SwiftUI body 재평가
                          .position(x, y+hopY) + linear(0.16s) 보간
```

`@State private var mongles` 배열을 매 0.16초마다 `step()`이 in-place로 갱신하면, SwiftUI가 0.16초 동안 선형 보간으로 자연스럽게 이동시킨다. 그 위에 hop, 벽 반사, 충돌 정지, idle 휴식 같은 보조 규칙이 얹혀 있다.

## 1. `MongleCharacter` — 캐릭터 1마리의 상태 (L1138~1161)

```swift
public struct MongleCharacter: Identifiable {
    public let id = UUID()
    public let name: String
    public var color: Color
    public var hasAnswered: Bool
    public var hasSkipped: Bool
    public var position: CGPoint
    public var targetPosition: CGPoint
    public var overlapCounter: Int = 0
    public var stepCount: Int = 0
    public var restFramesLeft: Int = 0
    public var hopY: CGFloat = 0
}
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1139 | `public let id = UUID()` | `Identifiable` 준수용 고유 식별자. `ForEach` 의 키. |
| L1140 | `let name: String` | 멤버 이름. 외부 `MongleMember` 와 매칭하는 키이기도 하다. |
| L1141 | `var color: Color` | 캐릭터 색. 외부 멤버 상태 변경 시 in-place로 업데이트됨. |
| L1142 | `var hasAnswered: Bool` | 오늘 질문 답변 여부. 뱃지 표시용. |
| L1143 | `var hasSkipped: Bool` | 오늘 질문 스킵 여부. 뱃지 표시용. |
| L1144 | `var position: CGPoint` | **현재 위치**. SwiftUI `.position` 으로 그려지는 좌표. |
| L1145 | `var targetPosition: CGPoint` | **다음 목표 좌표**. 캐릭터는 항상 이쪽으로 직선 이동한다. |
| L1146 | `var overlapCounter: Int = 0` | 충돌이 **연속으로 몇 프레임 지속됐는지** 카운트. 데드락 해소용. |
| L1147 | `var stepCount: Int = 0` | 누적 이동 스텝 수. **두 가지 용도**: (1) hop 위상 계산 (2) SwiftUI 애니메이션 트리거. |
| L1148 | `var restFramesLeft: Int = 0` | 휴식 잔여 프레임. >0이면 그 프레임은 모든 처리를 건너뛴다. |
| L1151 | `var hopY: CGFloat = 0` | 위로 튕기는 hop 오프셋 **캐시**. body 재평가마다 sin/abs 반복 호출을 막기 위해 step() 시점에 미리 계산해둔다. |

> ⓘ `var` 들이 모두 mutable인 이유: `[MongleCharacter]` 배열이 `@State` 라서 `mongles[i].xxx = ...` 형태로 in-place 갱신이 가능해야 한다.

## 2. `MongleSceneView` 의 시뮬레이션 상수 (L1334~1340)

```swift
private let stepSize: CGFloat = 2.0
/// 0.16s = 6.25Hz. 이전 0.12s(8.3Hz) 대비 부하 약 25% 감소하면서 시각적 자연스러움 유지.
private let interval: TimeInterval = 0.16
private let collisionRadius: CGFloat = 76
private let targetThreshold: CGFloat = 12
private let wallPadding: CGFloat = 50
private let overlapLimit: Int = 10
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1334 | `stepSize: CGFloat = 2.0` | **1 프레임당 이동 픽셀**. 6.25Hz × 2pt = **12.5pt/s** 의 등속 이동. |
| L1336 | `interval: TimeInterval = 0.16` | 시뮬레이션 틱 주기. 6.25Hz. SwiftUI 보간 duration도 동일하게 묶어 쓴다. |
| L1337 | `collisionRadius: CGFloat = 76` | 두 캐릭터의 거리가 76pt 미만이면 **충돌**로 본다. 캐릭터의 시각적 지름 + 약간의 여유. |
| L1338 | `targetThreshold: CGFloat = 12` | 거리가 12pt 미만이면 target에 **도달**한 것으로 본다. (한 스텝(2pt)보다 충분히 커서 진동 없이 안정적으로 종료) |
| L1339 | `wallPadding: CGFloat = 50` | 활동 영역의 안쪽 여백. 이 padding 안쪽 사각형이 캐릭터의 실제 활동 영역. |
| L1340 | `overlapLimit: Int = 10` | 충돌이 10프레임(≈1.6초) 이상 지속되면 **강제로 새 target** 을 줘서 데드락을 풀어준다. |

> ⓘ 이 6개 값이 시뮬레이션의 "느낌"을 결정한다. 캐릭터를 더 빠르게 하려면 `stepSize`↑ 또는 `interval`↓.

## 4. body 렌더링 (L1381~1462)

### 4.1 ZStack + ForEach 캐릭터 렌더 (L1383~1402)

```swift
ZStack {
    ForEach(mongles) { h in
        // hopY 는 step() 시점에 캐시됨 (body 재평가 시 sin/abs 반복 호출 제거)
        MongleView( /* … name/color/플래그/탭 콜백 … */ )
            .position(CGPoint(x: h.position.x, y: h.position.y + h.hopY))
            .animation(.linear(duration: interval), value: h.stepCount)
    }
}
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1384 | `ForEach(mongles) { h in` | 배열 순회. `MongleCharacter: Identifiable` 이라서 id 클로저가 필요 없다. |
| L1393 | `isCurrentUser: currentUserName != nil && h.name == currentUserName` | 본인 캐릭터인지 판정. 본인이면 다른 탭 동작. |
| L1400 | `.position(CGPoint(x: h.position.x, y: h.position.y + h.hopY))` | **이게 캐릭터를 "그리는" 핵심**. 시뮬레이션이 만든 좌표에 hop 오프셋을 더해 화면에 배치. |
| L1401 | `.animation(.linear(duration: interval), value: h.stepCount)` | **부드러운 이동의 비밀**. `stepCount` 가 1 증가할 때마다 0.16초 동안 새 `position` 까지 선형 보간. 6.25Hz의 띄엄띄엄한 갱신을 부드럽게 보이게 만든다. 휴식/충돌 시 stepCount 가 안 늘어나기 때문에 자연스럽게 멈춘다. |

### 4.2 onAppear / onChange(of: geo.size) (L1404~1414)

```swift
.onAppear {
    if geo.size.width > 0, geo.size.height > 0 {
        if mongles.isEmpty { initMongles(size: geo.size) }
        startTimer(size: geo.size)
    }
}
.onChange(of: geo.size) { _, newSize in
    guard newSize.width > 0, newSize.height > 0 else { return }
    if mongles.isEmpty { initMongles(size: newSize) }
    if timer == nil { startTimer(size: newSize) }
}
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1405 | `if geo.size.width > 0, geo.size.height > 0` | **첫 레이아웃 패스에서 size가 0인 케이스를 방어**. `randomPos` 가 음수 범위를 만들지 않도록. |
| L1406 | `if mongles.isEmpty { initMongles(...) }` | 첫 진입에서만 초기화. 재진입(예: 백그라운드→복귀)에는 기존 배열 유지. |
| L1407 | `startTimer(size: geo.size)` | 0.16초 타이머 가동 시작. |
| L1410 | `.onChange(of: geo.size)` | 회전, 분할 화면 등으로 size가 변할 때 fallback. onAppear가 size 0으로 호출됐던 케이스를 여기서 만회. |
| L1413 | `if timer == nil { startTimer(...) }` | 이미 가동 중이면 중복 시작 안 함. |

### 4.3 onChange(of: scenePhase) (L1415~1425)

```swift
.onChange(of: scenePhase) { _, newPhase in
    // 백그라운드/비활성 진입 시 Timer 정지, 복귀 시 재시작
    if newPhase == .active {
        if geo.size.width > 0, geo.size.height > 0, timer == nil {
            startTimer(size: geo.size)
        }
    } else {
        timer?.invalidate()
        timer = nil
    }
}
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1417 | `if newPhase == .active` | 포그라운드 복귀. |
| L1418 | `timer == nil` 가드 | 이미 돌아가고 있으면 중복 가동 안 함. |
| L1421~1424 | `else { timer?.invalidate(); timer = nil }` | `.inactive`/`.background` 진입 시 즉시 정지. **백그라운드에서 CPU 0**. |

> ⓘ 주석에서 강조한 의도: `NavigationStack` 으로 push 되면 SwiftUI가 onDisappear를 늦게 호출하거나 안 부르는 경우가 있다. 그 케이스에서도 scenePhase로 안전하게 멈춘다.

### 4.4 onChange(of: members) (L1430~1455) — 외부 상태 동기화

```swift
.onChange(of: members) { oldMembers, newMembers in
    guard geo.size.width > 0, geo.size.height > 0 else { return }

    let oldNames = Set(oldMembers.map { $0.name })
    let newNames = Set(newMembers.map { $0.name })
    if oldNames != newNames {
        initMongles(size: geo.size)
        return
    }

    let lookup = Dictionary(uniqueKeysWithValues: newMembers.map { ($0.name, $0) })
    for i in mongles.indices {
        guard let member = lookup[mongles[i].name] else { continue }
        if mongles[i].hasAnswered != member.hasAnswered { mongles[i].hasAnswered = member.hasAnswered }
        if mongles[i].hasSkipped  != member.hasSkipped  { mongles[i].hasSkipped  = member.hasSkipped }
        if mongles[i].color       != member.color       { mongles[i].color       = member.color }
    }
}
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1431 | `guard geo.size... else { return }` | size가 0이면 init 호출이 어차피 의미 없으니 종료. |
| L1434~1435 | `Set(...map { $0.name })` 두 개 비교 | **이름 집합 변화** = 가족 구성원 자체가 바뀐 것. |
| L1436~1439 | `if oldNames != newNames { initMongles; return }` | 시뮬레이션 자체를 새로 시작. 위치/타겟/카운터 전부 리셋. |
| L1442 | `Dictionary(uniqueKeysWithValues: ...)` | 이름→멤버 인덱스. 아래 루프에서 **O(1) 매칭**. |
| L1443~1454 | `for i in mongles.indices` + 필드별 변경시에만 대입 | 이름이 같다면 **위치/타겟은 보존**하고 답변/스킵/색만 in-place 업데이트. SwiftUI 의 view diff가 최소한만 invalidate 되게 함. |

> ⓘ 주석에서: 이전엔 4개의 `onChange` + 매번 `members.map` 임시배열 4개 + nested O(N²) linear search였다. 1개로 통합하면서 dict 기반으로 바꾼 최적화 흔적.

### 4.5 onDisappear (L1456~1459)

```swift
.onDisappear {

}
```
- view가 사라지면 타이머 무조건 정지. `scenePhase` 처리와 더해 이중 안전망.

## 5. 초기 배치 — `initMongles(size:)` (L1464~1484)

```swift
private func initMongles(size: CGSize) {
    guard size.width > 0, size.height > 0 else { return }
    var placed: [CGPoint] = []
    mongles = effectiveMembers.map { member in
        var pos = randomPos(size: size)
        for _ in 0..<30 {
            let overlaps = placed.contains { hypot(pos.x - $0.x, pos.y - $0.y) < collisionRadius }
            if !overlaps { break }
            pos = randomPos(size: size)
        }
        placed.append(pos)
        return MongleCharacter(
            name: member.name,
            color: member.color,
            hasAnswered: member.hasAnswered,
            hasSkipped: member.hasSkipped,
            position: pos,
            targetPosition: randomPos(size: size)
        )
    }
}
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1465 | `guard size.width > 0, size.height > 0 else { return }` | size가 0일 때 호출되면 randomPos가 잘못된 범위를 만든다. 방어. |
| L1466 | `var placed: [CGPoint] = []` | **이미 배치된 좌표 누적용**. 다음 캐릭터의 겹침 검사 대상. |
| L1467 | `mongles = effectiveMembers.map { member in` | 입력 멤버 1명당 한 마리 생성. `effectiveMembers` 는 외부 멤버가 비었으면 기본 5명 폴백. |
| L1468 | `var pos = randomPos(size: size)` | 첫 후보 좌표. |
| L1469 | `for _ in 0..<30` | **최대 30번 재시도**. 무한 루프 방지. |
| L1470 | `placed.contains { hypot(...) < collisionRadius }` | **이미 배치된 좌표들 중 76pt 안에 있는 게 있는지** 검사. |
| L1471 | `if !overlaps { break }` | 안 겹치면 즉시 채택. |
| L1472 | `pos = randomPos(size: size)` | 겹치면 다시 뽑아 재시도. |
| L1474 | `placed.append(pos)` | 30번 안에 못 찾았어도 그냥 마지막 후보를 사용. **시뮬레이션이 init에서 멈추지 않게** 하기 위한 양보. |
| L1475~1482 | `return MongleCharacter(...)` | 최종 캐릭터 생성. |
| L1481 | `targetPosition: randomPos(size: size)` | 시작 좌표와 무관하게 **첫 목표를 미리 부여**. 이래야 step() 첫 호출부터 캐릭터가 곧장 움직인다. |

> ⓘ 30번 재시도 + 폴백 채택은 **"항상 끝난다"** 를 보장하는 패턴. 인원이 많거나 영역이 좁으면 약간 겹칠 수 있지만, 곧 step()의 충돌 로직이 풀어준다.

## 6. `randomPos` / `startTimer` (L1486~1498)

```swift
private func randomPos(size: CGSize) -> CGPoint {
    CGPoint(
        x: CGFloat.random(in: wallPadding...(size.width - wallPadding)),
        y: CGFloat.random(in: wallPadding...(size.height - wallPadding))
    )
}

private func startTimer(size: CGSize) {
    timer?.invalidate()
    timer = Timer.scheduledTimer(withTimeInterval: interval, repeats: true) { _ in
        step(size: size)
    }
}
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1488~1489 | `CGFloat.random(in: wallPadding...(size.width - wallPadding))` | **활동 영역 안쪽** 에서만 랜덤. 시작/타겟이 처음부터 벽 안에 있게 강제. |
| L1494 | `timer?.invalidate()` | 기존 타이머가 있으면 먼저 정지. **중복 가동 방지**. |
| L1495 | `Timer.scheduledTimer(withTimeInterval: 0.16, repeats: true)` | 0.16초마다 클로저 호출. |
| L1496 | `step(size: size)` | 호출 시점의 size를 캡처해 사용. (크기 변경 시 onChange에서 다시 startTimer 호출됨) |

## 7. 매 프레임 시뮬레이션 — `step(size:)` (L1500~1554) ⭐ 핵심

전체 함수를 6 구간 (a)~(f) 로 쪼개서 **한 줄씩** 본다.

```swift
private func step(size: CGSize) {
    for i in mongles.indices {
        // (a) 휴식
        // (b) 목표 도달 판정
        // (c) 이동
        // (d) 벽 감지
        // (e) 충돌 감지
        // (f) 정상 이동 commit + hop
    }
}
```

### (a) 휴식 처리 (L1502~1508)

```swift
if mongles[i].restFramesLeft > 0 {
    mongles[i].restFramesLeft -= 1
    if mongles[i].restFramesLeft == 0 {
        mongles[i].targetPosition = randomPos(size: size)
    }
    continue
}
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1501 | `for i in mongles.indices` | 각 캐릭터 in-place 갱신을 위해 인덱스 순회. (`for c in mongles` 로는 mutate 불가) |
| L1502 | `if mongles[i].restFramesLeft > 0` | 휴식 중인지 확인. |
| L1503 | `mongles[i].restFramesLeft -= 1` | 1 프레임 차감. |
| L1504~1506 | `if ... == 0 { targetPosition = randomPos() }` | **0이 되는 그 프레임에 미리 새 목표 부여**. 다음 프레임부터 자연스럽게 출발. |
| L1507 | `continue` | 휴식 중이면 위치/충돌/벽 판정 모두 스킵. 캐릭터는 그대로 서 있다. |

### (b) 목표 도달 판정 (L1510~1523)

```swift
var pos = mongles[i].position
let target = mongles[i].targetPosition
let dx = target.x - pos.x
let dy = target.y - pos.y
let dist = hypot(dx, dy)

if dist < targetThreshold {
    if Bool.random() {
        mongles[i].restFramesLeft = Int.random(in: 10...50)
    } else {
        mongles[i].targetPosition = randomPos(size: size)
    }
    continue
}
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1510 | `var pos = mongles[i].position` | 로컬 변수로 복사. 이번 프레임에 적용할지/말지 마지막에 결정한다. |
| L1511 | `let target = mongles[i].targetPosition` | 현재 목표 스냅샷. |
| L1512 | `let dx = target.x - pos.x` | x 방향 거리(부호 포함). |
| L1513 | `let dy = target.y - pos.y` | y 방향 거리(부호 포함). |
| L1514 | `let dist = hypot(dx, dy)` | 유클리드 거리. **이후 단위벡터 계산에 다시 쓰이므로 한 번만 계산**. |
| L1516 | `if dist < targetThreshold` | 12pt 이내면 도달로 간주. (한 스텝(2pt)보다 충분히 큰 값이라 진동 없음) |
| L1517 | `if Bool.random()` | 50/50 분기. 도달 시 다양성 연출. |
| L1518 | `mongles[i].restFramesLeft = Int.random(in: 10...50)` | 휴식 길이 = 10~50 프레임 = **약 1.6 ~ 8 초**. |
| L1520 | `mongles[i].targetPosition = randomPos(size: size)` | 즉시 다음 목표로 출발. |
| L1522 | `continue` | 도달 처리한 프레임은 위치 갱신 안 함. |

### (c) 이동: 단위벡터 × stepSize (L1525~1526)

```swift
pos.x += (dx / dist) * stepSize
pos.y += (dy / dist) * stepSize
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1525 | `pos.x += (dx / dist) * stepSize` | `(dx/dist)` = x 방향 단위벡터 성분. 거기에 2pt 곱해 더함. |
| L1526 | `pos.y += (dy / dist) * stepSize` | y 방향도 동일. **모든 캐릭터가 같은 속도로 직선 이동**. |

> ⓘ 가속도/관성 없음. 단순 직선이라 코드가 짧고 예측 가능하다. 캐릭터마다 다른 속도가 필요하면 여기에 곱하기를 추가하면 된다.

### (d) 벽 감지 (L1528~1533)

```swift
if pos.x < wallPadding || pos.x > size.width - wallPadding ||
    pos.y < wallPadding || pos.y > size.height - wallPadding {
    pos.x = min(max(pos.x, wallPadding), size.width - wallPadding)
    pos.y = min(max(pos.y, wallPadding), size.height - wallPadding)
    mongles[i].targetPosition = randomPos(size: size)
}
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1528 | `if pos.x < wallPadding \|\| pos.x > size.width - wallPadding \|\|` | 좌/우 벽 이탈 검사. |
| L1529 | `pos.y < wallPadding \|\| pos.y > size.height - wallPadding` | 상/하 벽 이탈 검사. 4면 중 하나라도 벗어났으면 블록 진입. |
| L1530 | `pos.x = min(max(pos.x, wallPadding), size.width - wallPadding)` | **clamp**. `max(pos.x, wallPadding)` 로 하한 보정 후, `min(..., 우측한계)` 로 상한 보정. 벽을 뚫고 그려지지 않게. |
| L1531 | `pos.y = min(max(pos.y, wallPadding), size.height - wallPadding)` | y도 동일. |
| L1532 | `mongles[i].targetPosition = randomPos(size: size)` | **핵심**: 새 목표를 부여해서 다음 프레임부터 다른 방향으로 향하게 만든다. 이게 없으면 벽에 붙어서 계속 같은 방향으로 가려 한다. |

> ⓘ 물리적 "튕김"이 아니라 "clamp + 재타겟"으로 처리한다. 결과적으로 벽 근처에서 **방향 전환** 처럼 보인다. 또한 이 블록을 통과한 후에도 `pos`는 commit되지 않은 상태이며, 그대로 (e) 충돌 검사로 넘어간다.

### (e) 충돌 감지 — 겹침 방지의 핵심 (L1535~1547)

```swift
let collides = mongles.indices.contains { j in
    guard j != i else { return false }
    return hypot(pos.x - mongles[j].position.x,
                 pos.y - mongles[j].position.y) < collisionRadius
}
if collides {
    mongles[i].overlapCounter += 1
    if mongles[i].overlapCounter >= overlapLimit {
        mongles[i].targetPosition = randomPos(size: size)
        mongles[i].overlapCounter = 0
    }
    continue
}
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1535 | `let collides = mongles.indices.contains { j in ... }` | 자신 외 모든 캐릭터와 거리 검사. `contains` 는 첫 매치에서 즉시 종료. |
| L1536 | `guard j != i else { return false }` | 자기 자신은 제외. |
| L1537~1538 | `hypot(pos.x - mongles[j].x, pos.y - mongles[j].y) < collisionRadius` | **이번 프레임에 이동하려는 새 좌표 `pos`** 기준으로 다른 캐릭터의 **현재 위치** 와 거리 비교. 즉 "이 자리로 가면 겹친다" 를 사전에 검사. |
| L1540 | `if collides {` | 충돌이면 |
| L1541 | `mongles[i].overlapCounter += 1` | 충돌 지속 카운트 증가. |
| L1542 | `if mongles[i].overlapCounter >= overlapLimit` | 10프레임(≈1.6초) 이상 막혀 있다면 |
| L1543 | `mongles[i].targetPosition = randomPos(size: size)` | **데드락 해소**: 새 random target 부여. |
| L1544 | `mongles[i].overlapCounter = 0` | 카운터 리셋. 새 목표 잡았으니 다시 0부터. |
| L1546 | `continue` | **결정적 한 줄**: `pos` 를 commit 안 함. 캐릭터는 **그 프레임 그대로 정지**. 밀어내거나 우회 X. |

> ⓘ N=가족 멤버(보통 ≤ 8)이므로 O(N²) 풀 페어 검사가 무시 가능. 단순 정지 방식이라 디버깅도 쉽고, 데드락은 위 데드락 해소 로직으로 풀린다.

### (f) 정상 이동 commit + hop (L1549~1553)

```swift
mongles[i].overlapCounter = 0
mongles[i].stepCount += 1
mongles[i].position = pos
// hopY 캐싱: body 재평가마다 sin/abs 반복 호출하던 것을 step 시점으로 이동
mongles[i].hopY = -abs(sin(CGFloat(mongles[i].stepCount) * .pi / 5.0)) * 12
```

| 라인 | 코드 | 해석 |
|---|---|---|
| L1549 | `mongles[i].overlapCounter = 0` | 충돌 없이 통과했으니 카운터 리셋. (충돌이 끝난 직후 다시 출발하는 캐릭터의 카운터를 0으로) |
| L1550 | `mongles[i].stepCount += 1` | **이게 SwiftUI 애니메이션 트리거**. `.animation(value: stepCount)` 가 fire되어 보간이 시작된다. |
| L1551 | `mongles[i].position = pos` | 이번 프레임 새 좌표 commit. SwiftUI가 0.16초 동안 부드럽게 보간. |
| L1553 | `mongles[i].hopY = -abs(sin(CGFloat(mongles[i].stepCount) * .pi / 5.0)) * 12` | **hop 효과 캐싱**. 자세한 분해는 아래. |

#### hopY 식 자세히 분해

```
hopY = -abs(sin(stepCount · π / 5)) · 12
       └────┬─────┘  └────┬────┘   └┬┘
            │             │         │
            │             │         └─ 진폭 12pt
            │             └─────────── 5스텝(=0.8초)이 한 hop 주기
            └───────────────────────── 부호: -. SwiftUI는 y↓ 가 양수이므로,
                                       위로 솟구치는 효과를 위해 음수로.
        abs(...): 항상 위쪽으로만 튕김 (sin이 음수일 때 땅에 박히지 않게)
```

- `stepCount * π / 5` → 5스텝마다 위상이 π씩 진행. `|sin|` 의 한 아치가 한 hop.
- 결과적으로 **5 frames(=0.8s) 마다 위로 솟구쳤다 내려오는 hop**.
- 단, 충돌/휴식 시 stepCount 가 안 늘어나므로 hop도 자연스럽게 멈춘다.

## 9. 자주 받는 질문 / 튜닝 포인트

- **캐릭터를 더 빠르게**: `stepSize` 를 키우거나 `interval` 을 줄인다. 단 너무 빠르면 충돌 검사를 건너뛰는 듯한 감이 든다.
- **충돌 반경을 다르게**: `collisionRadius` 만 바꾸면 됨. 단 init 검사와 step 검사가 같은 값을 쓰므로 동기화돼 있다는 점 의식.
- **N 이 커진다면**: 현재 O(N²) 충돌 검사이지만 가족 단위(보통 ≤ 8)에선 충분. 100명 단위가 되면 quad-tree나 spatial hash 도입 고려.
- **캐릭터마다 속도/색상 변주**: `MongleCharacter` 에 `speed` 필드 추가 후 (c) 구간 곱하기에 반영.
- **히트박스 모양 변경**: 현재 원형(`hypot`). 사각형이면 `abs(dx) < r && abs(dy) < r` 로 바꾸면 더 가벼움.
