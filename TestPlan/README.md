# 테스트 계획 (Test Plan)

> 작성일: 2026-03-23
> 대상 프로젝트: iOS (FamTree) + 서버 (FamTreeServer)

---

## 현황 분석

| 구분 | 상태 |
|------|------|
| iOS Domain | 테스트 타겟 존재, 실질적 테스트 없음 |
| iOS MongleData | 테스트 없음 |
| iOS MongleFeatures | 테스트 없음 |
| 서버 | Jest 설정 존재, 테스트 파일 없음 |

현재 시뮬레이터로 수동 확인하는 방식은 회귀 버그 탐지에 취약하며,
특히 비즈니스 로직(하트 경제, 질문 배정, 그룹 초대)의 신뢰성을 보장하기 어렵습니다.

---

## 테스트 우선순위 기준

1. **버그 발생 시 파급 범위가 큰 영역** (인증, 질문 배정, 하트 경제)
2. **순수 함수/로직이 많아 테스트 비용 대비 효과가 높은 영역** (서비스 레이어, TCA 리듀서, DTO 매퍼)
3. **회귀 위험이 높은 영역** (그룹 선택 플로우, 알림 배지 로직)
